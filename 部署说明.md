# 内网服务器执行手册（同事直接照做）

本手册用于在公司内网服务器上完成以下目标：

1. 后端启动成功
2. 前端管理台启动成功
3. 生成任务可落库（`saved_count > 0`）
4. 离线 Judge 任务可完成（`status=completed`）

---

## 0. 前置约束（必须满足）

- 代码目录：本项目根目录（含 `admin_api.py`、`admin-web/`、`tools/`）。
- Node 版本：`20.x`（推荐 `20.19.0`）。
- Python：`3.8+`。
- `填写您的Key.txt` 必须在项目根目录存在且为有效 Key（不会通过 git 分发）。

---

## 1. 拉取代码与进入目录

```bash
git pull
cd /path/to/boxue-ai-exam-generator
```

---

## 2. 准备 Key 文件（关键）

> 注意：`填写您的Key.txt` 被 `.gitignore` 忽略，必须在服务器本机配置。

```bash
cp 填写您的Key.txt.example 填写您的Key.txt
chmod 600 填写您的Key.txt
```

编辑 `填写您的Key.txt`，至少填一组可用配置（建议 AIT）：

```text
AIT_API_KEY=你的真实密钥
AIT_BASE_URL=https://openapi-ait.ke.com/v1
AIT_MODEL=deepseek-chat
```

---

## 3. 安装依赖（统一入口）

```bash
# 必须先切到 Node 20.x（若已是 20.x 可跳过）
nvm install 20.19.0
nvm use 20.19.0

# 一键安装 Python + 前端依赖，并校验 vite
bash tools/bootstrap_server.sh
```

预期：

- 输出包含 `node=v20.x`
- 输出包含 `[bootstrap] done`
- 无报错退出

---

## 4. 启动后端与前端

### 方式 A：常规重启脚本

```bash
bash tools/restart_admin.sh
```

预期：

- 后端监听：`127.0.0.1:8600`
- 前端监听：`127.0.0.1:8522`
- 脚本末尾输出两条 URL 与两条日志路径

### 方式 B：手动启动（排障用）

```bash
# 终端1
.venv/bin/python admin_api.py

# 终端2
npm --prefix admin-web run dev -- --host 127.0.0.1 --port 8522
```

---

## 5. 启动健康检查

```bash
curl -s -m 3 -H "X-System-User: admin" http://127.0.0.1:8600/
curl -s -m 3 -H "X-System-User: admin" http://127.0.0.1:8600/api/tenants
```

预期：

- 第1条返回 `ok=true`
- 第2条返回租户列表（`items`）

---

## 6. 生成链路验证（必须跑）

> 先在管理后台确认目标租户已经完成：
> - 有切片文件 `knowledge_slices_*.jsonl`
> - 教材已映射
> - 切片审核 + 映射审核至少有一条同时完成
> - 该教材已设为生效

创建 1 题 smoke 任务（示例租户 `wh`）：

```bash
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/generate/tasks" \
  -d '{
    "task_name":"smoke_generate_2026_01",
    "num_questions":1,
    "question_type":"单选题",
    "difficulty":"中等",
    "generation_mode":"随机"
  }'
```

拿到 `task_id` 后查询：

```bash
curl -s -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/generate/tasks/<task_id>"
```

通过标准：

- `status=completed`
- `saved_count > 0`
- `run_id` 非空

---

## 7. Judge 链路验证（必须跑）

以第 6 步生成的 `run_id` 为输入：

```bash
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/judge/tasks" \
  -d '{
    "task_name":"smoke_judge_2026_01",
    "run_id":"<run_id>"
  }'
```

拿到 `judge_task_id` 后查询：

```bash
curl -s -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/judge/tasks/<judge_task_id>"
```

通过标准：

- `status=completed`
- `error_count=0`（允许少量错误但需人工评估）
- 对应 run 在 `qa/runs` 中出现 Judge 指标

---

## 8. 上线前真实 LLM 门禁（推荐）

```bash
python3 run_release_gate_real_llm.py --tenant-id wh
```

报告位置：

```text
tmp/release_gate_real_llm_report.json
```

通过标准：

- 脚本退出码为 `0`
- 报告内 `ok=true`

---

## 9. 常见报错与处理

### 报错：`未配置可用 API Key，请检查 填写您的Key.txt`

处理：

1. 确认文件在项目根目录：`ls 填写您的Key.txt`
2. 确认不是占位符内容（必须真实 Key）
3. 重启服务后重试

### 报错：`unsupported node=vXX, require major 20.x`

处理：

```bash
nvm install 20.19.0
nvm use 20.19.0
bash tools/bootstrap_server.sh
```

### 报错：`审核记录与当前切片版本不一致`

处理：

1. 在后台重新执行该教材版本的切片/映射审核
2. 确保切片与映射均已审核通过
3. 重新发起生成任务

### 报错：`该 run 无落库题（saved=true），无法运行 Judge`

处理：

1. 先让生成任务产出并落库（`saved_count>0`）
2. 再以该 `run_id` 发起 Judge

---

## 10. 交付验收口径（给业务方）

以下四项全部满足，才算“可上线”：

1. 服务可启动：`8600/8522` 正常监听
2. 生成可用：1题 smoke 任务 `saved_count>0`
3. Judge 可用：对应 run 的 Judge 任务 `completed`
4. 门禁通过：`run_release_gate_real_llm.py` 返回 PASS
