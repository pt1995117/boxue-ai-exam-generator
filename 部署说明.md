# 内网服务器执行手册（同事直接照做）

本手册用于在公司内网服务器上完成以下目标：

1. 后端启动成功
2. 前端管理台启动成功
3. 生成任务可落库（`saved_count > 0`）
4. 离线 Judge 任务可完成（`status=completed`）

## 0. 前置约束

- 代码目录包含 `admin_api.py`、`admin-web/`、`tools/`
- Node 版本：`20.x`
- Python：`3.8+`
- 项目根目录必须存在 `填写您的Key.txt`
- 映射模型目录建议已准备好：`models/bge-small-zh-v1.5/`

## 1. 拉取代码

```bash
git pull
cd /path/to/boxue-ai-exam-generator
```

## 2. 准备 Key 文件

```bash
cp 填写您的Key.txt.example 填写您的Key.txt
chmod 600 填写您的Key.txt
```

至少填一组可用配置：

```text
AIT_API_KEY=你的真实密钥
AIT_BASE_URL=https://openapi-ait.ke.com/v1
AIT_MODEL=deepseek-chat
```

## 3. 安装依赖

```bash
nvm install 20.19.0
nvm use 20.19.0
bash tools/bootstrap_server.sh
```

预期输出包含：

- `node=v20.x`
- `[bootstrap] done`

## 4. 启动后端与前端

推荐：

```bash
bash tools/restart_admin.sh
```

手动排障：

```bash
.venv/bin/python admin_api.py
npm --prefix admin-web run dev -- --host 127.0.0.1 --port 8522
```

## 5. 健康检查

```bash
curl -s -m 3 -H "X-System-User: admin" http://127.0.0.1:8600/
curl -s -m 3 -H "X-System-User: admin" http://127.0.0.1:8600/api/tenants
```

## 6. 生成链路验证

先确认目标租户已有：

- 切片文件 `knowledge_slices_*.jsonl`
- 参考题映射
- 至少一条切片完成切片审核和映射审核
- 教材已设为生效

创建 1 题 smoke 任务：

```bash
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/generate/tasks" \
  -d '{
    "task_name":"smoke_generate_2026_01",
    "num_questions":1,
    "question_type":"单选题",
    "difficulty":"中等",
    "generation_mode":"随机"
  }'
```

通过标准：

- `status=completed`
- `saved_count > 0`
- `run_id` 非空

## 7. Judge 链路验证

```bash
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-System-User: admin" \
  "http://127.0.0.1:8600/api/wh/judge/tasks" \
  -d '{
    "task_name":"smoke_judge_2026_01",
    "run_id":"<run_id>"
  }'
```

通过标准：

- `status=completed`
- 对应 run 能看到 Judge 结果
