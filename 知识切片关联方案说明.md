# 知识切片到母题自动关联方案

## 方案概述

本方案实现了知识切片（Knowledge Slices）到母题（Historical Questions）的自动关联，目标实现**80%全自动关联率**。

## 核心策略

### 策略1：反向索引（Reverse Index）
- **原理**：利用现有的 `question_knowledge_mapping.json` 构建反向索引
- **优势**：直接复用已有映射，准确率100%
- **适用场景**：已有母题→知识切片映射的情况

### 策略2：路径匹配（Path Matching）
- **原理**：匹配母题的篇/章/节信息是否在知识切片路径中
- **评分规则**：
  - 篇匹配：+0.3
  - 章匹配：+0.3
  - 节匹配：+0.4
  - 总分≥0.3则纳入候选
- **适用场景**：母题有明确的篇/章/节标注

### 策略3：考点精确匹配（Exact Kaodian Match）
- **原理**：母题的"考点"字段是否在知识切片路径中
- **置信度**：0.85
- **适用场景**：考点命名规范，与知识路径一致

### 策略4：TF-IDF语义相似度（TF-IDF Similarity）
- **原理**：计算知识切片内容与母题（考点+题干）的语义相似度
- **评分公式**：
  ```
  combined_score = 0.6 × TF-IDF相似度 + 0.4 × 关键词覆盖率
  ```
- **自动通过条件**：
  - `combined_score >= 0.4` (0.6×0.25 + 0.4×0.25)
  - `TF-IDF相似度 >= 0.25`
  - `关键词覆盖率 >= 0.25`
- **适用场景**：语义相关但路径不完全匹配的情况

### 策略5：LLM重排序（LLM Rerank）
- **原理**：对于低置信度（<0.2）且无匹配的情况，使用大模型从候选池中选择
- **触发条件**：
  - 自动匹配置信度 < 0.2
  - 当前无任何匹配
  - 有API密钥配置
- **适用场景**：复杂语义匹配，需要理解上下文

## 工作流程

```
1. 加载数据
   ├─ 知识切片 (bot_knowledge_base.jsonl)
   ├─ 母题 (存量房买卖母卷ABCD.xls)
   └─ 现有映射 (question_knowledge_mapping.json)

2. 构建索引
   ├─ TF-IDF向量化（母题）
   └─ 反向索引（知识切片→母题列表）

3. 对每个知识切片：
   ├─ 策略1：反向索引匹配
   ├─ 策略2：路径匹配
   ├─ 策略3：考点匹配
   ├─ 策略4：TF-IDF相似度匹配
   └─ 策略5：LLM重排序（低置信度时）

4. 输出结果
   └─ knowledge_question_mapping.json
```

## 输出格式

```json
{
  "0": {
    "完整路径": "第一篇 > 第一章 > ...",
    "掌握程度": "掌握",

        "confidence": 0.85,

```

## 使用方法

### 1. 安装依赖

```bash
pip install pandas scikit-learn openpyxl openai
```

### 2. 配置API密钥

在 `填写您的Key.txt` 中配置：
```
DEEPSEEK_API_KEY=your_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-reasoner
```

### 3. 运行脚本

```bash
python map_knowledge_to_questions.py
```

### 4. 查看结果

- 映射文件：`knowledge_question_mapping.json`
- 控制台输出：统计信息、方法分布、置信度分布

## 参数调优

### 自动通过阈值

```python
AUTO_PASS_TARGET = 0.8  # 目标80%自动关联
MIN_AUTO_PASS_TFIDF = 0.25  # TF-IDF最低阈值
MIN_AUTO_PASS_COVERAGE = 0.25  # 关键词覆盖率最低阈值
AUTO_PASS_WEIGHT_TFIDF = 0.6  # TF-IDF权重
AUTO_PASS_WEIGHT_COVERAGE = 0.4  # 覆盖率权重
```

### LLM触发阈值

```python
LLM_CONFIDENCE_THRESHOLD = 0.2  # 低于此值触发LLM
TOP_N_CANDIDATES = 10  # LLM候选数量
```

## 性能优化建议

1. **批量处理**：可以分批处理知识切片，避免内存溢出
2. **缓存TF-IDF**：如果知识切片不变，可以缓存TF-IDF矩阵
3. **并行处理**：可以多进程处理不同知识切片
4. **LLM限流**：对于大量数据，建议添加LLM调用限流
