# 智能出题系统 - 任务规格文档 (Task Specification)

## 1. 文档信息

- **文档版本**: v1.0
- **创建日期**: 2025-01-27

- **维护者**: 搏学考试团队

## 2. 文档说明

本文档将技术设计文档（TDD）中的功能点拆分为具体的实现任务（Task），每个任务包含：
- **任务描述**: 明确的任务目标和范围
- **实现要求**: 具体的技术实现要求
- **验收标准 (DoD)**: 可直接转化为TDD断言或自动化测试脚本的验收标准
- **关键测试点**: 必须测试的核心功能点

## 3. 核心功能任务

### Task 1: Router节点实现

**任务描述**: 实现Router节点，负责分析知识点类型、提取掌握程度、决定派发专家，并支持重路由检测与专有名词锁定清单生成。

**实现要求**:
- 文件: `exam_graph.py` - `router_node()` 函数
- 特征检测：公式、表格、列表
- 专有名词识别：基于 `房地产行业专有名词新.xlsx` 做关键词召回，并结合切片语义语境判定是否为同一概念
- 路由决策：计算类→CalculatorAgent，法律类→LegalAgent，综合类→GeneralAgent
- 题型推荐：公式→单选，列表→多选，表格→判断
- 重路由检测：检测并清理旧状态
- 术语状态传递：将命中术语写入 `state['term_locks']` 传递到后续节点

**验收标准 (DoD)**:
1. ✅ Router节点能正确检测知识点中的公式、表格、列表特征
2. ✅ Router节点能根据特征正确派发到calculator_node或specialist_node
3. ✅ Router节点能提取并传递掌握程度信息
4. ✅ Router节点能检测重路由（retry_count > 0）并清理旧状态
5. ✅ Router节点能生成题型推荐（recommended_type）
6. ✅ Router节点输出的agent_name必须是有效的（CalculatorAgent/LegalAgent/GeneralAgent）
7. ✅ Router节点能生成专有名词锁定清单（term_locks）

### Task 1b: 切片结构化展示（UI）

**任务描述**: UI 中按切片结构化字段原顺序展示内容，例题/解析不得嵌入表格。

**实现要求**:
- 文件: `app.py`
- 使用结构化内容字段渲染（`context_before/tables/context_after/examples/formulas/images`）
- 渲染顺序必须与切片原始结构一致
- `examples` 作为独立区块展示，禁止混入表格渲染

**验收标准 (DoD)**:
1. ✅ 切片展示顺序与结构化字段顺序一致
2. ✅ 例题/解析独立区块展示，不与表格混排
3. ✅ 表格内容仅包含表格字段

**关键测试点**:
- **测试点1.1**: 包含公式的知识点 → 应派发到CalculatorAgent
  - 断言: `agent_name == "CalculatorAgent"`
  - 测试数据: 知识点包含`formulas`字段
- **测试点1.2**: 包含表格的知识点 → 应推荐判断题
  - 断言: `router_details['recommended_type'] == "判断题"`
  - 测试数据: 知识点包含`tables`字段
- **测试点1.3**: 重路由检测 → 应清理旧状态但保留prev_final_json和prev_critic_feedback
  - 断言: `state['draft'] is None` 且 `state['prev_final_json'] is not None`
  - 测试数据: `retry_count > 0` 且存在`prev_final_json`
- **测试点1.4**: 掌握程度提取 → 应从kb_chunk正确提取
  - 断言: `router_details['mastery'] == kb_chunk.get('掌握程度', '未知')`
  - 测试数据: kb_chunk包含`掌握程度`字段
- **测试点1.5**: 专有名词语义命中 → 仅“关键词命中+语义一致”才进入term_locks
  - 断言: `state['term_locks']` 仅包含语义语境一致的术语
  - 测试数据: 构造“同词不同义”与“同词同义”两类切片

### Task 2: Specialist节点实现

**任务描述**: 实现Specialist节点，处理法律、政策、综合知识类题目，根据知识点和母题范例生成初稿。

**实现要求**:
- 文件: `exam_graph.py` - `specialist_node()` 函数
- 支持教材原题优先机制（builtin_examples优先）
- 支持掌握程度约束（影响题目复杂度设计）
- 严格遵循题型、模式、难度约束
- 数据重构：禁止直接照搬原文案例中的具体数据
- **题型格式规范**：判断/单选/多选结构一致，且括号使用中文括号“（ ）”、括号前后无空格、括号内为全角空格
- 命中 `term_locks` 的专有名词必须原词输出，禁止同义替换/缩写替换
- **遣词造句与指代一致**：题干注意主谓搭配与指代一致，避免指代对象错误导致语义偏差
- **判断题定义类例外**：行为/说法类判断题用【XX做法正确/错误】；定义类判断题（含属于/是指/定义/概念）可用其他表述，不强制该模板

**验收标准 (DoD)**:
1. ✅ Specialist节点能优先使用教材原题（builtin_examples）
2. ✅ Specialist节点能在提示词中包含掌握程度约束
3. ✅ Specialist节点生成的初稿符合指定的题型（single/multi/judge）
4. ✅ Specialist节点生成的初稿符合指定的出题筛选条件（基础概念/理解记忆、实战应用/推演、随机）
5. ✅ Specialist节点生成的初稿符合指定的难度范围
6. ✅ Specialist节点输出的draft字段是有效的JSON格式
7. ✅ 判断/选择题括号格式为中文括号“（ ）”，括号前后无空格、括号内为全角空格
8. ✅ Specialist节点命中术语保持原词不改写
9. ✅ Specialist节点提示词包含“遣词造句与指代一致”要求
10. ✅ Specialist节点判断题规范包含“定义类判断题例外”说明

**关键测试点**:
- **测试点2.1**: 教材原题优先 → 应优先使用builtin_examples
  - 断言: `examples[0]` 来自 `kb_chunk['结构化内容']['examples']`（如果存在）
  - 测试数据: kb_chunk包含`结构化内容.examples`
- **测试点2.2**: 掌握程度约束 → 提示词中应包含掌握程度信息
  - 断言: 生成的prompt包含`掌握程度要求为: 【{mastery}】`
  - 测试数据: kb_chunk包含`掌握程度`字段（了解/熟悉/掌握）
- **测试点2.3**: 题型约束 → 生成的题目应符合指定题型
  - 断言: 单选题有4个选项且答案长度为1，多选题答案长度>1，判断题选项为["正确","错误"]
  - 测试数据: `question_type = "单选题"/"多选题"/"判断题"`
- **测试点2.4**: 筛选条件约束 → 三档筛选条件应严格执行
  - 断言: 基础概念/理解记忆包含"不强制业务场景"；实战应用/推演包含"必须关联业务场景"；随机模式包含"自动选择两者之一"
  - 测试数据: `generation_mode = "基础概念/理解记忆"/"实战应用/推演"/"随机"`
- **测试点2.5**: 括号规范 → 判断/选择题括号格式必须为 `（ ）`
  - 断言: 题干占位括号为 `（ ）`，括号前后无空格
  - 测试数据: 含答案占位括号的题干

### Task 3: Calculator节点实现

**任务描述**: 实现Calculator节点，动态生成Python计算代码，执行计算获取结果，并根据计算结果生成初稿。

**实现要求**:
- 文件: `exam_graph.py` - `calculator_node()` 函数
- **模型智能切换策略**：
  - 读取 `.gpt_rate_limit.txt` 文件，检测 GPT 限流状态
  - 如果距离上次调用 < 10 秒且需等待 > 5 秒，切换到 `deepseek-chat`
  - 将实际使用的模型存入 `state['calculator_model_used']`（用于 UI 显示）
  - 打印切换日志：`⚠️ GPT 限流中（需等待 {wait}s），切换到 Deepseek`
- 动态生成Python计算代码
- 在沙箱环境中安全执行代码
- 支持掌握程度约束
- 参数提取：必须提取具体数值，禁止描述性文字
- **题型格式规范**：生成题目需符合指定题型结构，括号统一为中文括号“（ ）”
- 命中 `term_locks` 的专有名词必须按原词出现在题干/选项/解析
- **遣词造句与指代一致**：题干注意主谓搭配与指代一致，避免指代对象错误导致语义偏差
- **判断题定义类例外**：行为/说法类判断题用【XX做法正确/错误】；定义类判断题（含属于/是指/定义/概念）可用其他表述，不强制该模板

**验收标准 (DoD)**:
1. ✅ Calculator节点检测GPT限流状态，限流等待>5秒时自动切换Deepseek
2. ✅ Calculator节点将实际使用的模型存入state['calculator_model_used']
3. ✅ Calculator节点能根据知识点动态生成Python计算代码
4. ✅ Calculator节点能在沙箱环境中安全执行代码
5. ✅ Calculator节点能在提示词中包含掌握程度约束
6. ✅ Calculator节点生成的代码能正确提取具体数值参数
7. ✅ Calculator节点执行代码时能捕获执行异常与导入错误
8. ✅ Calculator节点输出的calculation_code、calculation_result字段有效
9. ✅ 判断/选择题括号格式为中文括号“（ ）”，括号前后无空格、括号内为全角空格
10. ✅ Calculator节点命中术语保持原词不改写
11. ✅ Calculator节点提示词包含“遣词造句与指代一致”要求
12. ✅ Calculator节点判断题规范包含“定义类判断题例外”说明

**关键测试点**:
- **测试点3.1**: 代码生成 → 应生成有效的Python代码
  - 断言: `calculation_code` 是有效的Python代码字符串，包含`result =`赋值
  - 测试数据: 知识点包含计算规则（如契税计算）

  - 断言: 提取的参数是数字类型（int/float），不是字符串描述
  - 测试数据: 题干包含"80平方米"、"1560元"等具体数值
- **测试点3.5**: 掌握程度约束 → 提示词中应包含掌握程度信息

### Task 4: Writer节点实现

**任务描述**: 实现Writer节点，将初稿转化为标准JSON格式，验证并调整难度值，清洗格式。

**实现要求**:
- 文件: `exam_graph.py` - `writer_node()` 函数
- **题型修改策略**：
  - **随机模式**：如果 `config['question_type'] == "随机"`，Writer **不允许**修改专家节点生成的题型，必须保持原样
  - **指定题型模式**：如果 `config['question_type']` 为具体题型（单选/多选/判断），Writer 才需要校验并强制修改为指定题型
- **题型状态传递**：将确定的题型保存到 `state['current_question_type']`，供后续节点使用
- 难度验证：检查并调整难度值到指定范围
- 格式清洗：去除选项前缀（A./A、/A:等）
- **格式硬修复与校验（润色后）**：
  - 先完成 LLM 润色
  - 再用代码统一修复题干/选项括号格式、括号位置、句号位置
  - 去除选项末尾标点（句号/分号/逗号等）
  - 校验答案格式（单选=单字母，多选=多字母或列表，判断=A/B）
  - 判断题强制选项为“正确/错误”，单选题强制 4 个选项，多选题至少 4 个选项
  - 修复失败只记录问题，不阻断流程，仍流入 Critic
- **人名规范代码校验与改写（Writer 内完成）**：
  - 禁止使用“姓+女士/先生”“小+姓氏”等称谓
  - Writer 必须在润色后执行代码级检测并触发整体改写（显式提示人名违规；不得直接替换为“某某”）
  - 改写后进入 Critic
- **硬规则独立校验器（Writer Validate）**：
  - Writer Validate 调用独立硬规则校验器输出 `ValidationReport.issues`
  - 校验范围：年份约束、表格/图片禁用、单引号禁用、兜底选项禁用、数值选项升序、题干/选项/解析长度阈值
  - 表格/图片禁用需与离线 Judge `REQUIREMENTS.md 4.5` 对齐，且覆盖**题干/选项/解析**：
    - 图片：`![...](...)`、`<img>`、`.png/.jpg/.jpeg/.gif`
    - 表格：Markdown 表格行（`|` + `---` 行）与 `<table>`
  - Writer 仅基于 issues 做最小修复，不改题意

- **动态意图约束（仅特定题型触发）**：
  - 仅当题干被识别为 `ENUMERATION`（完整清单/资料包括）或 `EXCLUSION`（不需要/不包括）时，才触发强校验
  - 允许无标签题；无标签题不触发动态约束
  - 禁止 “SELECTION（单点选择/最XX/关键）” 类题型；Writer 与 Critic 必须规避
  - 语义覆盖允许改写，但**专有名词库中的词禁止同义/语义等价处理**
  - 专有名词命中规则：关键词召回 + 语义语境一致（双条件）
  - 专有名词库主文件：`房地产行业专有名词新.xlsx`（可预处理缓存为 `教材提取专有名词.txt`）
- **扩展切片上下文（Writer 需使用）**：
  - 基于当前知识片段的“完整路径”，找到上一级路径
  - 将上一级路径下**全部知识切片**拼接为上下文补充
  - 若存在参考母题（examples），基于母题题干+解析进行相似度检索，补充**相关知识切片**
- Writer 自检需同时基于【当前切片 + 上一级切片全集 + 相似切片】
- 出题节点（Specialist/Calculator/General）需基于【当前切片 + 上一级切片全集 + 相似切片】做**题干一致性自检**，若不一致必须输出**问题清单**给 Writer
  - 问题清单需明确指出：冲突维度、冲突点、建议调整方向（改背景/改条件/改答案）
  - Writer 必须据此修订题干与解析，并保持与自检问题一一对应的修复结果
- **遣词造句与指代一致**：润色时注意主谓搭配与指代一致，避免指代对象错误
- **判断题定义类例外**：润色判断题时，定义类（含属于/是指/定义/概念）可不强制【XX做法正确/错误】模板；行为/说法类仍应使用该模板
- 地理和时间约束检查
- **题型格式规范**：输出题目结构符合题型；括号统一为中文括号“（ ）”

**验收标准 (DoD)**:
1. ✅ Writer节点能将初稿转化为标准JSON格式（符合ExamQuestion模型）
2. ✅ Writer节点能验证难度值并在不符合范围时调整到范围中点
3. ✅ Writer节点能清洗选项前缀（去除A./A、/A:等）
4. ✅ 随机模式：Writer节点保持专家节点生成的题型不变
5. ✅ 指定题型模式：Writer节点强制修改为指定题型
6. ✅ Writer节点能将确定的题型保存到state['current_question_type']
7. ✅ Writer节点输出的final_json能通过Pydantic验证
8. ✅ Writer节点能检查并修正地理和时间逻辑问题
9. ✅ 判断/选择题括号格式为中文括号“（ ）”，括号前后无空格、括号内为全角空格
10. ✅ Writer节点在润色后完成格式硬修复与校验（括号位置/句号位置/答案格式）
11. ✅ 判断题题干以括号结尾，选择题题干以“（ ）。 ”结尾（无多余空格）
12. ✅ Writer节点对选项数量做强制规范（判断=2，单选=4，多选≥4）
13. ✅ Writer节点去除选项末尾标点（句号/分号/逗号等）
14. ✅ Writer节点对“姓+女士/先生/小+姓氏”进行代码级拦截并触发整体改写（不得直接替换为“某某”）
15. ✅ 动态意图约束仅对 `ENUMERATION/EXCLUSION` 触发；无标签题不触发
16. ✅ 禁止 “SELECTION（单点选择/最XX/关键）” 类题型
17. ✅ 语义覆盖允许改写但专有名词必须原文一致（来源：`房地产行业专有名词新.xlsx`）
