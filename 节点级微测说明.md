# 第一级：节点级微测

## 一、检索端微测：Hit Rate @ Top 3

### 1. 黄金基准集

- 文件：`golden_set_retrieval.csv`
- 列：`考点`，`预期片段`
- 可扩充到 50 条（或先用 18 条种子跑通）。`预期片段` 应为教材中该考点逻辑的代表性关键词（如：契税→「首套」「面积」；土地出让金→「成本价」「1%」）。

### 1b. AI 自动化：扩充 + 教材化 + 审计 + 切分建议（推荐）

```bash
python3 ai_golden_set_and_audit.py
```

该脚本会：

1. **挖掘考点**：从 `kb_data` 的 `完整路径` 中自动挖掘约 50 个考点；
2. **教材化预期片段**：对每个考点取对应教材片段，调用 LLM 抽取 1 个「典型表述」作为 `预期片段`，写入 `golden_set_retrieval.csv`（原文件备份为 `.bak`）；
3. **跑 Hit Rate @ Top 3**：调用 `run_retrieval_hit_rate.compute_hit_rate`；
4. **逻辑截断审计**：对每条未命中，用 LLM 判断是否逻辑截断、缺失什么、给出切分建议；
5. **汇总预处理/切分建议**：由 LLM 合并为 3–5 条总体建议；
6. **写报告**：`节点级微测报告.md`。

依赖：`填写您的Key.txt` 中 `OPENAI_*` 已配置；LLM 调用次数约 `目标考点数` + 未命中数 + 1，耗时视模型而定。

- 可选参数：`python3 ai_golden_set_and_audit.py [目标考点数] [--dry-run]`
  - 目标考点数：5–50，默认 50。例如 `15` 表示只处理 15 个考点（约 15 次 LLM 预期片段）。
  - `--dry-run`：跳过所有 LLM，用考点前 4 字作 dummy 预期、审计与汇总用占位文案；用于快速检查流程与 Hit 率，会覆盖 `golden_set_retrieval.csv`（原文件备份为 `.bak`，可自行还原）。

### 2. 仅跑 Hit 率（已有人工黄金集时）

```bash
python3 run_retrieval_hit_rate.py
```

- 逻辑：按 `考点` 在 `kb_data` 的 完整路径/核心内容 中做包含检索，取 Top 3；若任一 Top3 的 `核心内容` 包含 `预期片段` 则判为 Hit。
- 指标：**Hit Rate @ Top 3 = Hit 数 / 总样本数**。

### 3. 逻辑完整度审计

- 对 **未命中** 的样本做人工审阅：Top3 是否因 Chunking 被截断（例如只有「税率」没有「判定条件」）。
- 若截断多：调整教材的**预处理与逻辑语义分割**，再重跑 Hit Rate。

## 二、计算节点微测：工具执行率

### 1. 范围

- 从 `kb_data` 中按 契税、土地出让金、房龄、贷款、增值税、建筑面积、容积率、得房率、价差率 等关键词筛选出计算相关 chunk，最多 `limit` 个（默认 20）。

### 2. 跑测

```bash
# 快速冒烟（5 条，约 1–2 分钟）
python3 run_calculation_execution_rate.py 5

# 完整 20 条（约 5–10 分钟，视模型延迟）
python3 run_calculation_execution_rate.py 20

# 100 条（批量）
python3 run_calculation_execution_rate.py 100
```

- 逻辑：对每个 chunk 调用 CalculatorNode 的**计划步骤**（同 `exam_graph` 的 plan prompt），得到 `{ need_calculation, tool, params }`；若 `need_calculation=true` 且 `tool` 非空，则执行 `RealEstateCalculator.<tool>(**params)`，捕获异常。
- 指标：**执行成功率 = 成功次数 / 尝试执行次数**（目标 >95%）。
- 未触发计算（`need_calculation=false` 或 `tool=None`）的 chunk 记为「跳过」，不纳入成功率分母。

### 3. 归因（成功率 <95% 时）

- **幻觉**：`tool` 在 `RealEstateCalculator` 中不存在 → 强化 Prompt 与 `CALCULATION_GUIDE`，约束仅从给定函数中选择。
- **语法/参数**：`TypeError`、`KeyError` 等 → 约束 `params` 的类型与必填参数，必要时做 schema 校验或数值化。
- **逻辑**：能跑通但结果与教材不符 → 核对 `calculation_logic.py` 与教材规则。

## 三、落地与扩展

| 微测维度       | 脚本                         | 核心 KPI              | 改进方向                       |
|----------------|------------------------------|------------------------|--------------------------------|
| 检索节点       | `run_retrieval_hit_rate.py`  | Hit Rate @ Top 3      | 预处理、索引与检索策略         |
| 计算节点       | `run_calculation_execution_rate.py` | 执行成功率 >95% | 强化 Plan 的代码/参数约束      |
| 专家生成节点   | （可后续加）                 | Grounding / 事实一致性 | 信息不对称校验、语境与维度约束 |

- 若需「生成 Python 代码再沙箱执行」的独立微测，可另写脚本：让模型只输出代码块，在受限 `exec` 或子进程中运行，再统计执行成功率与归因。
