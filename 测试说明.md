# 批量题目生成测试说明

## 已创建的测试脚本

### 1. `generate_one_question.py` - 交互式单题生成（推荐）
**用途**: 每次运行生成一道题目，可以选择题目类型

**使用方法**:
```bash
python generate_one_question.py
```

**特点**:
- 交互式选择题目类型（计算题/非计算题/随机）
- 显示完整的生成过程和题目详情
- 可以多次运行生成不同题目

### 2. `run_batch_test.py` - 批量生成测试
**用途**: 一次性生成多道题目（3道计算题 + 3道非计算题）

**使用方法**:
```bash
python run_batch_test.py
```

**特点**:
- 自动生成6道题目（3道计算题 + 3道非计算题）
- 显示每道题的生成过程和结果汇总
- 适合快速测试系统功能

### 3. `quick_batch_test.py` - 快速批量测试
**用途**: 简化版批量测试

**使用方法**:
```bash
python quick_batch_test.py
```

### 4. `batch_test_questions.py` - 完整批量测试
**用途**: 最完整的批量测试，包含详细统计和JSON导出

**使用方法**:
```bash
python batch_test_questions.py
```

**特点**:
- 生成更多题目（5道计算题 + 5道非计算题）
- 保存结果到JSON文件
- 包含详细的统计信息

## 推荐测试流程

### 快速测试（推荐）
```bash
# 1. 先测试单题生成，确保系统正常
python generate_one_question.py

# 2. 选择"1"生成计算题，观察是否走 CalculatorAgent
# 3. 选择"2"生成非计算题，观察是否走 WriterAgent
# 4. 多次运行，测试不同知识点
```

### 批量测试
```bash
# 生成6道题目（3计算 + 3非计算）
python run_batch_test.py
```

## 测试要点

### 1. 计算题测试
- ✅ 应该走 `CalculatorAgent` 路由
- ✅ 应该生成 Python 计算代码
- ✅ Critic 应该使用 Qwen2.5-Math（如果配置了）
- ✅ 题目应该包含具体的人物、地点、场景（场景化参数注入）
- ✅ 题目应该将隐式条件转化为具体场景（如"张先生30岁，贷款25年"）

### 2. 非计算题测试
- ✅ 应该走 `WriterAgent` 路由
- ✅ 题目应该包含具体场景（避免"干巴巴"）
- ✅ 干扰项应该巧妙设计（利用除外条件）

### 3. 信息不对称校验测试
- ✅ Critic 应该进行"反向解题"
- ✅ 如果题目遗漏条件，应该被驳回
- ✅ Critic 应该能根据全量规则推导出唯一答案

### 4. 场景化参数注入测试
- ✅ 题目应该包含具体人物（如"张先生"）
- ✅ 题目应该包含具体地点（如"北京市朝阳区"）
- ✅ 题目应该将隐式规则转化为具体场景

## 使用 Streamlit UI 测试（可视化）

也可以使用现有的 Streamlit UI 进行可视化测试：

```bash
streamlit run app.py
```

在 UI 中：
1. 选择知识点
2. 选择题目类型
3. 点击生成
4. 观察整个生成流程的可视化

## 注意事项

1. **API Key**: 确保 `填写您的Key.txt` 中配置了正确的 API Key
2. **网络**: 确保能访问 DeepSeek API（如果使用 DeepSeek）
3. **时间**: 每道题目生成需要 30-60 秒，批量测试需要较长时间
4. **输出**: 如果终端输出有问题，可以重定向到文件：
   ```bash
   python run_batch_test.py > test_output.txt 2>&1
   ```

## 预期结果

### 成功的题目应该：
- ✅ 题干完整，包含具体场景
- ✅ 4个选项都有内容
- ✅ 正确答案在 A/B/C/D 中
- ✅ 解析完整（包含教材原文、试题分析、结论）
- ✅ 难度值在 0-1 之间
- ✅ 考点明确

### 计算题额外要求：
- ✅ 题干中包含具体数值（如"80平方米"、"1560元"）
- ✅ 场景化描述（如"张先生购买了一套位于北京市朝阳区的已购公房"）
- ✅ 隐式条件转化（如"房龄+贷款年限≤50年"转化为具体年龄和贷款年限）
