## Calculator Node 计算流程详解

```mermaid
flowchart TD
    Start([开始]) --> Examples[📚 照猫画虎<br/>获取母题范例]
    Examples --> Codegen[🧩 生成计算代码]
    
    Codegen --> Exec[🧮 执行计算代码]
    Exec --> Q1{是否需要<br/>多步计算?}
    Q1 -->|是| Step1[步骤1: 使用计算器<br/>计算中间结果<br/>如: 房龄]
    Q1 -->|否| Direct[直接计算<br/>最终结果]
    
    Step1 --> Step2[步骤2: 基于步骤1<br/>进一步计算<br/>如: 贷款年限 = 50 - 房龄]
    Step2 --> Step3[步骤3: 考虑其他因素<br/>如: 借款人年龄<br/>取最小值]
    Step3 --> Extract[📊 提取参数<br/>必须是具体数值<br/>不能是描述性文字]
    
    Direct --> Extract
    Extract --> Generate[✍️ 生成初稿<br/>根据 generation_mode<br/>灵活/严谨]
    
    Generate --> End([完成])
```

## Critic Node 验证流程详解

```mermaid
flowchart TD
    Start([开始]) --> Extract[📊 参数提取<br/>从题目中提取具体数值<br/>不能是描述性文字]
    
    Extract --> Check{能否<br/>提取参数?}
    Check -->|否| Skip[跳过计算器验证]
    Check -->|是| CodeCheck[✅ 代码规则校验<br/>是否符合教材公式与判定条件]
    
    CodeCheck -->|通过| Calc[🔧 执行计算代码]
    CodeCheck -->|不通过| FailFast[❌ 直接判 Fail]
    
    Calc --> StepAnalysis[✅ 计算步骤分析<br/>理解计算器可能只是<br/>解决一个步骤]
    
    StepAnalysis --> Q1{计算器结果<br/>是最终答案?}
    Q1 -->|是| DirectVerify[直接对比<br/>计算结果和题目答案]
    Q1 -->|否| MultiStep[验证完整计算过程<br/>步骤1: 计算器结果<br/>步骤2: 基于步骤1的计算<br/>步骤3: 考虑其他因素]
    
    DirectVerify --> AnswerCheck[验证答案正确性]
    MultiStep --> AnswerCheck
    Skip --> AnswerCheck
    FailFast --> AnswerCheck
    
    AnswerCheck --> ExplanationCheck[验证解析逻辑性<br/>如果涉及多步计算<br/>检查是否说明所有步骤]
    
    ExplanationCheck --> Decision{验证结果}
    Decision -->|通过| Pass[✅ 通过]
    Decision -->|需修复| Retry{retry_count}
    
    Retry -->|≥3| SelfHeal[🛡️ 自愈输出]
    Retry -->|<3| IssueType{问题类型}
    
    IssueType -->|轻微问题| Fixer[Fixer Node<br/>修复]
    IssueType -->|严重问题| Router[Router Node<br/>重新路由]
    
    Fixer --> Start
    Router --> Start
    Pass --> End([✅ 输出题目])
    SelfHeal --> End
    
    style Start fill:#e3f2fd,stroke:#1976d2,stroke-width:2px
    style Extract fill:#fff3e0,stroke:#f57c00,stroke-width:2px
    style Calc fill:#f3e5f5,stroke:#9c27b0,stroke-width:2px
```

## 节点功能说明

| 节点 | 功能 | 关键特性 |
|------|------|----------|
| **Router Node** | 路由决策 | 分析知识点、计算相关度、提取掌握程度 |
| **Calculator Node** | 计算题生成 | 生成计算代码、执行计算、计算步骤分析、参数提取 |
| **Specialist Node** | 法律/综合题生成 | 照猫画虎、生成初稿 |
| **Writer Node** | 格式化 | 标准化 JSON 输出 |
| **Critic Node** | 质量验证 | 参数提取、代码规则校验、计算步骤验证、答案验证、解析验证 |
| **Fixer Node** | 错误修复 | 本地修复、循环回到 Critic |
| **自愈机制** | 容错保护 | retry_count≥3 时自动输出 |

## 系统优势

1. 🎯 **质量保障**: 多层验证确保题目准确性
2. 🔄 **自适应修复**: 根据问题严重程度选择修复策略
3. 🛡️ **容错能力**: 完善的错误处理和重试机制
4. 📊 **可观测性**: 完整的日志记录每个决策节点
5. 🧮 **智能计算**: 理解计算步骤，支持多步计算验证
6. 🎨 **灵活出题**: 支持灵活/严谨两种模式，适应不同场景
7. 🎯 **精准参数**: 严格的参数提取规则，确保计算准确性
