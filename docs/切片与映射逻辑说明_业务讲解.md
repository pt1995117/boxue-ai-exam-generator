# 切片与映射逻辑说明（业务讲解版）

> 面向业务讲解，附带关键技术备注。  
> 文档版本：2026-03

---

## 一、整体关系与流程

```
教材上传 → 切片生成 → 切片核对(人工审核) → 上传母题库 → 映射生成 → 映射核对(人工确认) → 设为生效 → AI出题
```

| 阶段 | 业务含义 | 产出物 | 状态字段 |
|------|----------|--------|----------|
| 切片 | 把教材按章节拆成可出题的知识点单元 | `knowledge_slices_<版本>.jsonl` | `slice_status` |
| 映射 | 把每个知识点与历史母题建立关联 | `knowledge_question_mapping_<版本>.json` | `mapping_status` |
| 生效 | 切片+映射都成功后，才能设为出题教材 | - | `status=effective` |

**业务规则（当前实现）**：
- 切片成功 → 可在「切片核对」筛选到该教材
- 映射成功 → 可在「映射核对」筛选到该教材
- 生效教材 → 才在「AI出题」中可选

## 二、切片逻辑

### 2.1 业务目标

把 Word 教材按**层级结构**拆成若干「知识切片」，每个切片对应一个可独立出题的知识点，包含：
- 完整路径（篇 > 章 > 节 > 一、二、三…）
- 正文、表格、图片解析、公式、例题等结构化内容

### 2.2 处理流程（简化）

```
docx 文档
  → 解析段落/标题/表格/图片（python-docx）
  → 识别标题层级（正则：第X篇/章/节、一、二、（一）、（1））
  → 构建路径栈，按最小标题起新切片
  → 正文/表格归并到当前切片
  → 图片调用多模态大模型解析
  → 路径修复、附录公式回填、过滤目录
  → 输出 JSONL
```

### 2.3 切片输出结构（单条）

```json
{
  "完整路径": "第二篇 签前服务 > 第一章 房屋信息 > 第一节 房源信息概述 > 一、房源信息",
  "掌握程度": "了解",
  "结构化内容": {
    "context_before": "正文前置内容",
    "context_after": "正文后置内容",
    "tables": ["Markdown/HTML 表格"],
    "images": [{"image_id":"...", "analysis":"图片解析文本", "contains_table":false}],
    "formulas": ["公式1", "公式2"],
    "examples": ["【例】..."]
  },
  "metadata": {"类型":"自动组装", "表格索引":0, "图片索引":1}
}
```

### 2.4 技术要点

| 环节 | 技术实现 | 备注 |
|------|----------|------|
| 文档解析 | `python-docx` + XML 遍历 | 按 body 顺序读取段落、表格 |
| 标题识别 | 正则匹配 | 篇/章/节/一、/（一）/（1） 等 6 级 |
| 路径构建 | `path_stack` 栈 | 遇新标题覆盖下级，保证层级正确 |
| 表格渲染 | 自研 `_render_table_preserve_merge` | 无合并→Markdown；有合并→HTML rowspan/colspan |
| 图片解析 | 多模态大模型 | `analyze_image_with_qwen_vl`，Ark/AIT/OpenAI 兼容 |
| 路径修复 | `repair_flattened_paths` | 防止（一）等子级丢父级路径 |
| 附录公式 | `_assign_formulas_to_slices` | 从「计算公式汇总表」提取并回填到相关切片 |
| 目录过滤 | `is_toc_slice` | 过滤目录页，不产出切片 |

**切片阶段不涉及**：BGE、文本 LLM（仅图片用多模态）。

## 三、映射逻辑

### 3.1 业务目标

为每个知识切片找到与之相关的**历史母题**，建立「切片 ↔ 母题」映射，供出题时参考题型、难度和表述风格。

输入：
- 切片：`knowledge_slices_<版本>.jsonl`
- 母题库：历史考题 Excel（含篇/章/节/考点、题干、选项、解析）

输出：
- `knowledge_question_mapping_<版本>.json`：`{ slice_idx: { 完整路径, matched_questions: [...] } }`

### 3.2 映射策略（优先级从高到低）

| 策略 | 业务含义 | 置信度 | 技术实现 |
|------|----------|--------|----------|
| **Strategy 1: Reverse_Index** | 复用已有映射文件 | 1.0 | 若存在 `question_knowledge_mapping` 反向索引，直接命中 |
| **Strategy 2: GPS_FullPath** | 路径完全包含匹配 | 0.95/0.90/0.85 | 母题「篇+章+节+考点」与切片「完整路径」做包含/前缀匹配 |
| **Strategy 3: Statute_Collision** | 法条硬碰撞 | 0.88 | 从切片和母题中抽取《XX法》第X条，法条一致则匹配，再经 BGE 精修 |
| **Strategy 4: BGE_Vector** | 语义向量检索 | =BGE 分数 | BGE 对切片+母题文本做向量相似度，>0.75 自动通过 |
| **Strategy 5: LLM_Logic** | 大模型逻辑重排 | 0.80~0.90 | BGE 分数在 0.5~0.75 时，交给 LLM 做专家逻辑重排 |

### 3.3 策略 4、5 的阈值逻辑

```
BGE 分数 > 0.75  → 自动通过（BGE_Vector）
0.5 < BGE ≤ 0.75 → LLM 重排（LLM_Logic）
BGE ≤ 0.5        → 若无其他策略命中，可做 LLM fallback
```

### 3.4 后处理：每题 Top1

映射完成后会做**每题严格 Top1** 过滤：一道母题只保留置信度最高的一个切片，避免一题多切片的混乱。

### 3.5 技术要点

| 环节 | 技术实现 | 备注 |
|------|----------|------|
| BGE 模型 | `BAAI/bge-small-zh-v1.5` | sentence-transformers，本地向量化 |
| 切片向量文本 | `完整路径 + context_before/after + tables + examples` | 路径放前作为强锚点 |
| 母题向量文本 | `标准化路径 + 题干 + 选项 + 解析` | 与切片同空间 |
| LLM 调用 | OpenAI 兼容 API | 默认 deepseek-chat，可配 AIT/OPENAI |
| 路径标准化 | `normalize_path_dehydration` | 去「第X篇/章/节」、同义词替换 |
| 法条抽取 | `extract_legal_references` | 正则抽《XX》第X条 |

## 四、状态流转与生效规则

### 4.1 教材状态

| 状态 | 含义 |
|------|------|
| `slicing` | 切片脚本执行中 |
| `ready_for_review` | 切片成功，待映射或待设生效 |
| `effective` | 当前生效教材，可用于 AI 出题 |
| `archived` | 已下线 |

### 4.2 切片 / 映射状态

| 字段 | 取值 | 含义 |
|------|------|------|
| `slice_status` | success / failed / running / pending | 切片是否成功 |
| `mapping_status` | success / failed / running / pending | 映射是否成功 |

### 4.3 生效前置条件（后端强制校验）

设为生效前必须同时满足：
- `slice_status === success` 且切片文件存在
- `mapping_status === success` 且映射文件存在

上传教材后**不再自动生效**，需先完成映射，再手动「设为生效」。

## 七、业务讲解话术参考

**切片**：  
> 我们把教材按「篇-章-节-小节」拆成一块块知识点，每块叫一个切片。表格、图片也会自动解析成文字，方便后续出题时引用。

**映射**：  
> 映射是把每个切片和历史考题（母题）关联起来。我们先用路径、法条等规则匹配，匹配不到的用 BGE 语义检索，再交给大模型做一次逻辑把关，尽量保证「这道母题确实考的是这个知识点」。

**生效**：  
> 只有切片成功、映射也成功的教材，才能设为生效。生效后才会在 AI 出题里出现，避免用半成品教材出题。
