# 教材生效前置条件

本文档单独说明一个容易误解的规则：教材版本并不是“切片成功 + 映射成功”后就能直接设为生效。

## 1. 生效的真实前置条件

一个教材版本要设为生效，必须同时满足以下 5 条：

1. 教材版本存在
2. `slice_status=success`
3. `mapping_status=success`
4. 切片文件和映射文件都实际存在
5. 至少存在 1 条切片，同时满足：
   - 切片核对已完成
   - 映射确认已完成

## 2. 为什么不是“切片成功就能生效”

因为切片成功只代表：

- 技术上完成了文本切分
- 不代表切片内容一定正确
- 不代表切片层级、图片解析、切片顺序已经过人工确认

如果跳过人工核对，后续出题可能直接建立在错误切片上。

## 3. 为什么不是“映射成功就能生效”

因为映射成功只代表：

- 系统生成了一份切片到母题的关联结果
- 不代表这些关联一定符合业务预期
- 也不代表母题信息已经补全到可审核状态

如果跳过映射确认，可能会把错误母题当作出题参考样本。

## 4. 为什么只要求“至少 1 条双审核完成”

当前设计不是要求“全量切片都审核完成”才可生效，而是要求：

- 至少有 1 条切片同时完成切片核对和映射确认

这样设计的平衡点是：

- 不把上线门槛设得过高
- 但也不允许完全没有任何人工确认就上线

换句话说，这是一个最小人工验收门槛。

## 5. 前端如何判断“当前版本可生效”

教材列表接口会返回以下字段：

- `can_set_effective`
  - 是否可设为生效教材

- `dual_review_slice_count`
  - 已完成“双审核”的切片数量

- `effective_block_reason`
  - 如果不能生效，给出阻塞原因

## 6. 常见阻塞原因

### 6.1 切片未成功

表现：

- `slice_status != success`

影响：

- 直接不能生效

### 6.2 映射未成功

表现：

- `mapping_status != success`

影响：

- 直接不能生效

### 6.3 切片或映射文件缺失

表现：

- 状态看起来是成功，但物理文件不存在

影响：

- 后端会拒绝生效
- 列表接口也会尝试做状态自愈

### 6.4 双审核切片数为 0

表现：

- `dual_review_slice_count = 0`
- `can_set_effective = false`

影响：

- 返回 `MATERIAL_REVIEW_NOT_READY`

## 7. 推荐业务操作顺序

推荐的最小闭环：

1. 上传教材
2. 等待切片成功
3. 上传参考题并完成映射
4. 选至少 1 条关键切片做切片核对
5. 对应映射做确认
6. 再设为生效教材

推荐做法：

- 优先审核最关键、最常出题、最容易出错的切片
- 不要为了“尽快可用”而完全跳过人工审核

## 8. 重切片后的注意事项

一旦执行重切片：

- 旧切片审核会被清空
- 旧映射文件会被删除
- 旧映射审核也会失效

这意味着：

- 之前已经可生效的教材，重切片后可能重新变得不可生效
- 因为双审核条件需要重新建立

## 9. 适合在什么场景下设为生效

建议在以下条件下再设为生效：

- 当前教材是本城市最新要用的正式版本
- 关键切片已经做过人工抽检
- 映射质量已基本确认
- 计划立即用于出题任务

不建议：

- 教材刚上传完就立刻设生效
- 明知切片内容还没核过，只是想先“抢进度”

## 10. 一句话总结

教材生效不是纯技术动作，而是“技术处理完成 + 至少一轮人工验收完成”的联合门禁。
