# 管理后台页面操作手册

本文档给业务同学和运维同学一份最小可用的后台操作顺序说明，目标是能按页面跑通主链路。

## 1. 页面总览

当前后台主页面包括：

- 工作台
- 资源上传
- 切片核对
- 映射确认
- AI出题
- 出题模板
- 质量评估
- Judge任务
- 版本管理
- 题库
- 城市管理
- 全局Key配置

## 2. 使用前准备

进入后台后，先在顶部确认三件事：

1. 当前城市
2. 当前系统号
3. 是否需要填写 OIDC Token

如果城市切错，后面看到的教材、任务、题库都会是别的城市数据。

## 3. 标准业务顺序

建议按这个顺序操作：

1. 资源上传
2. 切片核对
3. 映射确认
4. 教材生效
5. 出题模板
6. AI出题
7. Judge任务
8. 质量评估
9. 版本管理
10. 题库

## 4. 工作台

用途：

- 看当前城市整体状态
- 快速确认教材、任务、题库是否有数据

适合做：

- 日常巡检
- 进入某城市后的第一眼确认

## 5. 资源上传

用途：

- 上传教材文件
- 查看教材版本列表
- 触发参考题上传、重切片、重映射、生效、归档等动作

典型流程：

1. 上传教材
2. 等待切片完成
3. 上传参考题
4. 等待映射完成
5. 准备进入审核页面

注意：

- 不是上传完成就能出题
- 教材必须经过切片审核与映射审核，且满足生效条件

## 6. 切片核对

用途：

- 人工确认切片质量
- 修改切片文本
- 查看切片图片与预览

建议做法：

1. 先按教材版本筛选
2. 优先处理 `pending`
3. 对明显错误切片做修改或驳回
4. 确认无误后通过审核

说明：

- 只有切片通过审核，才会进入模板和出题的“可用切片”集合

## 7. 映射确认

用途：

- 审核切片到母题的映射关系

建议做法：

1. 先过滤待审核项
2. 查看候选母题是否正确
3. 若候选母题不完整，按需要手工补题干/选项/解析
4. 再执行通过或驳回

注意：

- 映射通过前，系统会检查母题信息是否完整

## 8. 教材生效

教材能否生效，不只看“切片和映射都跑完”。

至少要满足：

- 当前教材下至少有一条切片已完成切片核对
- 且同一教材下至少有一条映射已完成映射确认

建议：

- 不要等到所有切片都审完才点生效
- 但也不要在几乎没审核的情况下过早生效

## 9. 出题模板

用途：

- 为已生效教材配置题量规划

典型动作：

1. 选择教材版本
2. 设置总题量
3. 配置掌握度比例
4. 配置知识路径前缀和占比
5. 保存模板

注意：

- 页面里只能选 `effective` 教材
- 模板保存成功不代表一定能出完题，还要看 approved 切片是否足够

## 10. AI出题

用途：

- 发起普通出题任务或模板出题任务

典型动作：

1. 选择普通任务或模板任务
2. 填写任务名
3. 选择教材/模板
4. 提交任务
5. 进入详情页观察 trace

详情页重点看：

- `status`
- `current_node`
- `generated_count`
- `saved_count`
- `critic_result`

## 11. Judge任务

用途：

- 对 run 中已落库题目执行离线 Judge

建议：

- 只对已稳定落库的 run 执行
- Judge 完成后再看质量评估和版本发布

## 12. 质量评估

用途：

- 看 overview、趋势、漂移、阈值、告警、发布报告

建议优先关注：

- `saved_count`
- `judge_pass_rate`
- `quality_score_avg`
- `cpvq`
- `error_call_rate`

## 13. 版本管理

用途：

- 基于已完成 Judge 的 run 发布版本

操作顺序：

1. 填版本号
2. 选可发布 run
3. 填发布说明
4. 视情况勾选 Git 提交
5. 执行发布

注意：

- 没有落库题的 run 不可发布
- 没跑 Judge 的 run 不可发布

## 14. 题库

用途：

- 查看最终题库
- AI 调优
- 手工编辑
- 删除
- 导出
- 手工加题

建议：

- 发布前后都可在题库页做抽检
- 导出时注意查看附带的来源切片和 Judge 字段

## 15. 城市管理

用途：

- 平台管理员管理城市与用户绑定

只适合：

- `platform_admin`

普通城市角色不应在这里做操作。

## 16. 全局Key配置

用途：

- 在线维护模型 Key、Git 凭据等全局配置

建议：

- 正式环境尽量由平台管理员统一维护
- 变更后做一次小流量验证，再让业务同学继续出题
