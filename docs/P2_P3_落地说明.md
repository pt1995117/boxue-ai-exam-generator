# P2/P3 落地说明（已实现）

## 1. P2 能力

### 1.1 OIDC 鉴权（Keycloak 兼容）
- 入口：`authn.py`
- 开关：`AUTH_MODE=oidc`（默认 `legacy`）
- 支持：
  - `Authorization: Bearer <token>` 验证
  - 角色/租户/权限从 token claim 解析
  - `OIDC_INSECURE_ALLOW_UNSIGNED=true` 本地联调模式
- 关键环境变量：
  - `OIDC_ISSUER_URL`
  - `OIDC_AUDIENCE`
  - `OIDC_JWKS_TTL_SEC`

### 1.2 PostgreSQL RLS 行级隔离
- 入口：`db_store.py`
- 行为：
  - PostgreSQL 下自动 `enable row level security`
  - 自动创建策略：`tenant_id = current_setting('app.tenant_id', true)`
  - 每次查询/写入前设置 `set_config('app.tenant_id', <tenant>, true)`

### 1.3 OpenTelemetry
- 入口：`observability.py`
- 开关：`OTEL_EXPORTER_OTLP_ENDPOINT` 非空时启用
- 已在 `admin_api.py` 关键接口埋点：
  - `api.slices`
  - `api.mappings`

## 2. P3 能力

### 2.1 限流
- 入口：`governance.py`
- 算法：滑动窗口（每用户+路径）
- 配置：`ADMIN_API_RATE_LIMIT_RPM`（默认 240）
- 超限返回：`429 + Retry-After`

### 2.2 熔断
- 入口：`governance.py`
- 配置：
  - `ADMIN_API_CIRCUIT_FAILURES`（默认 5）
  - `ADMIN_API_CIRCUIT_RECOVERY_SEC`（默认 30）
- 效果：连续失败打开熔断，恢复期返回 `503`

### 2.3 灰度
- 入口：`governance.py`
- 配置：
  - `ADMIN_API_CANARY_PERCENT`（0-100）
  - `ADMIN_API_CANARY_SALT`
- 强制指定：
  - 请求头 `X-Release-Channel: stable|canary`
- 响应头：
  - `X-Release-Channel`

### 2.4 API 元信息
- 新增：`GET /api/meta`
- 返回：鉴权模式、当前账号、角色、租户、灰度通道、功能开关状态

## 3. 前端支持
- 文件：`admin-web/src/services/api.js`
- 新增本地存储：
  - `auth_token`（OIDC）
  - 自动注入 `Authorization: Bearer ...`
- 页面新增 OIDC Token 输入：
  - `DashboardPage.jsx`
  - `SliceReviewPage.jsx`
