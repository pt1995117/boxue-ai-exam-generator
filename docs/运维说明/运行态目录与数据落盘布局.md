# 运行态目录与数据落盘布局

本文档说明当前项目的运行时目录、配置文件落点，以及哪些目录允许手工处理，哪些不建议直接改。

## 1. 总体原则

项目同时存在两套路径概念：

- 仓库内静态文件
- 运行时动态数据

默认情况下，运行时数据根目录位于：

- `.local/runtime`

缓存目录默认位于：

- `.local/cache`

如果环境变量覆盖了路径，则以环境变量为准：

- `BOXUE_RUNTIME_DIR`
- `BOXUE_CACHE_DIR`
- `BOXUE_KEY_FILE`
- `BOXUE_TENANT_USER_FILE`

代码参考：

- [runtime_paths.py](/Users/panting/Desktop/搏学考试/AI出题/runtime_paths.py#L6)

## 2. 关键根目录

| 路径 | 含义 |
| --- | --- |
| `REPO_ROOT` | 仓库根目录 |
| `REPO_DATA_DIR` | 仓库内 `data/` 目录 |
| `.local/runtime` | 默认运行态根目录 |
| `.local/runtime/data` | 默认租户运行数据目录 |
| `.local/runtime/config` | 默认运行态配置目录 |
| `.local/runtime/db/admin_p0.db` | 默认运行态 SQLite 数据库 |
| `.local/cache` | 默认缓存目录 |

## 3. 关键配置文件

### 3.1 Key 文件

优先级：

1. `BOXUE_KEY_FILE`
2. `.local/runtime/config/填写您的Key.txt`
3. 仓库根目录 `填写您的Key.txt`

说明：

- 运行时优先使用 runtime 下的 Key 文件
- 仓库根目录文件只作为回退

### 3.2 租户用户文件

优先级：

1. `BOXUE_TENANT_USER_FILE`
2. `.local/runtime/config/tenant_users.json`
3. 仓库根目录 `tenant_users.json`

代码参考：

- [runtime_paths.py](/Users/panting/Desktop/搏学考试/AI出题/runtime_paths.py#L33)

## 4. 租户数据目录

系统同时定义了两类租户目录：

- 仓库内：`data/{tenant_id}`
- 运行时：`.local/runtime/data/{tenant_id}`

说明：

- 老数据和部分业务文件仍可能在仓库内 `data/` 下
- 新运行态能力更推荐把动态产物收敛到 runtime 目录

代码参考：

- [runtime_paths.py](/Users/panting/Desktop/搏学考试/AI出题/runtime_paths.py#L45)

## 5. 常见运行态数据类型

按当前系统功能，大致会落这些数据：

- 教材上传文件
- 切片结果
- 切片图片
- 切片审核结果
- 映射结果
- 映射审核结果
- 出题任务快照
- QA runs / 发布记录 / 告警
- 题库文件
- 审计日志

其中最容易增长的通常是：

- 切片目录
- 图片目录
- 题库导出
- 任务快照与 trace

## 6. 可以手工查看但不建议直接改的内容

以下文件或目录适合排查，不建议人工直接编辑：

- 租户切片目录
- 映射结果文件
- 题库文件
- 出题任务快照
- QA 运行记录
- 审计日志

原因：

- 后端很多状态不是单文件决定，而是多个文件与数据库共同作用
- 手工改一个文件，容易造成页面状态和后端状态不一致

## 7. 可以安全清理的典型目录

通常可以清理或重建的内容：

- `.local/cache`
- 前端构建缓存
- 临时日志
- Mermaid/流程图生成的中间产物

但清理前仍建议确认当前没有在线任务。

## 8. 不建议直接删除的内容

不建议直接删：

- `admin_p0.db`
- 题库数据
- 审核结果
- 映射结果
- 教材上传原件
- 运行中的任务快照

如果这些文件需要迁移或清理，应优先通过脚本或业务接口完成。

## 9. 与 Git 工作区的边界

仓库自带了工区卫生检查脚本，会阻止把明显的运行态垃圾带进 Git。

重点拦截：

- `.local/`
- `logs/`
- `node_modules/`
- `data/*/audit/`
- `data/*/mapping/`
- `data/*/slices/`
- `data/*/bank/`
- 常见二进制产物如 `.db/.xlsx/.pdf`

代码参考：

- [tools/check_worktree_hygiene.sh](/Users/panting/Desktop/搏学考试/AI出题/tools/check_worktree_hygiene.sh)

## 10. 建议运维约束

- 运行环境统一设置 `BOXUE_RUNTIME_DIR`
- 不要把运行态文件长期写回仓库根目录
- Key 文件与租户用户文件优先放 runtime/config
- 数据迁移优先使用脚本，不要直接复制单个 JSON 试图“补状态”
