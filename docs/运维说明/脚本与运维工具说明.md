# 脚本与运维工具说明

本文档说明 `tools/` 目录下常见脚本的用途、典型使用时机，以及是否会修改数据。

## 1. 脚本清单

当前主要脚本如下：

- `tools/bootstrap_server.sh`
- `tools/restart_admin.sh`
- `tools/restart_admin_launchd.sh`
- `tools/check_worktree_hygiene.sh`
- `tools/install_git_hooks.sh`
- `tools/migrate_sqlite_to_postgres.py`
- `tools/recover_orphan_task_details.py`
- `tools/generate_admin_flowchart_jpg.py`

## 2. `bootstrap_server.sh`

用途：

- 初始化本地运行环境

主要动作：

- 检查 Node 和 npm
- 强制要求 Node 主版本为 `20.x`
- 创建 Python 虚拟环境
- 安装 Python 依赖
- 安装前端依赖
- 检查 Key 文件是否存在

注意：

- 如果 `填写您的Key.txt` 不存在，会直接报错退出，不会自动创建
- 这个脚本适合“首次准备环境”，不负责常驻重启

是否修改数据：

- 会安装依赖
- 会创建 `.venv`
- 会写运行目录与缓存目录

代码参考：

- [tools/bootstrap_server.sh](/Users/panting/Desktop/搏学考试/AI出题/tools/bootstrap_server.sh)

## 3. `restart_admin.sh`

用途：

- 本地快速重启前后端服务

主要动作：

- 先跑工区卫生检查
- 检查 Python/Flask 可用性
- 若 Key 文件缺失，则自动创建空文件
- 释放 `8600` 和 `8522` 端口
- 启动后端 `admin_api.py`
- 启动前端 Vite dev server
- 做监听与 HTTP 健康检查

注意：

- 这个脚本会真实 kill 占端口的进程
- 更适合开发机本地重启，不适合作为正式部署编排脚本

是否修改数据：

- 会写 `/tmp` 日志与 pid 文件
- 会创建空 Key 文件

代码参考：

- [tools/restart_admin.sh](/Users/panting/Desktop/搏学考试/AI出题/tools/restart_admin.sh)

## 4. `restart_admin_launchd.sh`

用途：

- 在 macOS 下通过 `launchd` 管理前后端服务

主要动作：

- 生成 `LaunchAgents` plist
- `launchctl bootstrap / kickstart`
- 释放端口
- 健康检查后端和前端

适用场景：

- 需要用系统级守护方式维持本机服务常驻时

注意：

- 这是偏本机运维脚本，不适合 Linux 服务器

## 5. `check_worktree_hygiene.sh`

用途：

- 检查 Git 工作区里是否混入运行态垃圾

重点拦截：

- `.local/`
- `logs/`
- `node_modules/`
- 租户运行数据目录
- `.db/.xlsx/.pdf` 等产物

适用场景：

- 提交前自检
- 重启脚本前置校验

是否修改数据：

- 不修改，只检查

## 6. `install_git_hooks.sh`

用途：

- 把仓库 Git hooks 指向 `.githooks`

主要动作：

- 设置 `core.hooksPath`
- 给 `pre-commit` 和 `pre-push` 加执行权限

是否修改数据：

- 会修改本地仓库 Git config

## 7. `migrate_sqlite_to_postgres.py`

用途：

- 把 SQLite 中的管理后台核心表迁移到 PostgreSQL

当前迁移表：

- `slice_review`
- `mapping_review`
- `material_registry`
- `audit_log`

支持参数：

- `--sqlite-path`
- `--target-database-url`
- `--dry-run`

适用场景：

- 数据库从 SQLite 切到 PostgreSQL
- 环境迁移前做数据搬运

是否修改数据：

- `dry-run` 只读
- 正常执行会写 PostgreSQL

代码参考：

- [tools/migrate_sqlite_to_postgres.py](/Users/panting/Desktop/搏学考试/AI出题/tools/migrate_sqlite_to_postgres.py)

## 8. `recover_orphan_task_details.py`

用途：

- 恢复“服务重启后被标记失败，但其实已有结果”的孤儿任务详情

主要逻辑：

- 扫描失败任务
- 识别错误消息是否属于服务重启型 orphan
- 尝试从 QA run 或题库回补题目
- 重建最小化 `items` 和 `process_trace`

适用场景：

- 服务重启后，历史任务列表还有记录，但详情页空了

是否修改数据：

- 会回写任务 JSONL 记录

代码参考：

- [tools/recover_orphan_task_details.py](/Users/panting/Desktop/搏学考试/AI出题/tools/recover_orphan_task_details.py)

## 9. `generate_admin_flowchart_jpg.py`

用途：

- 根据当前后台结构生成管理后台业务流程图 JPG

适用场景：

- 需要给业务或同事讲流程
- 生成静态流程图附件

依赖：

- `PIL`

是否修改数据：

- 只生成图片文件，不改业务数据

## 10. 使用建议

- 首次搭环境优先用 `bootstrap_server.sh`
- 日常本地重启优先用 `restart_admin.sh`
- 提交前可先跑 `check_worktree_hygiene.sh`
- 迁移数据库前先用 `migrate_sqlite_to_postgres.py --dry-run`
- 孤儿任务详情丢失时再用 `recover_orphan_task_details.py`
