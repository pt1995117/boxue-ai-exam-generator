# 题库字段字典

本文档说明题库接口中的常见 JSON 字段，以及 Excel 导出列的含义。

## 1. 题库对象定位

题库对象本质上是“最终可落库题目”的宽表结构。

它不仅存题干、选项、答案，也会额外补充：

- 来源任务
- 来源切片
- 关联切片
- 母题信息
- 离线 Judge 结果

## 2. JSON 常见字段

### 2.1 题目主体字段

| 字段 | 含义 |
| --- | --- |
| `题干` | 题目正文 |
| `选项1` ~ `选项8` | 选项内容，顺序对应 A~H |
| `正确答案` | 标准答案，通常是 `A/B/C/D` 或多选组合 |
| `解析` | 题目解析 |
| `题目类型` | 题型描述 |
| `难度值` | 难度数值 |

### 2.2 教材与知识点字段

| 字段 | 含义 |
| --- | --- |
| `教材版本ID` | 题目所属教材版本 |
| `一级知识点` ~ `四级知识点` | 分层知识点字段 |
| `来源路径` | 题目主要来源路径 |
| `切片原文` | 主来源切片文本 |

### 2.3 来源追踪字段

| 字段 | 含义 |
| --- | --- |
| `source_task_id` | 来源出题任务 ID |
| `source_task_name` | 来源出题任务名称 |
| `source_run_id` | 来源 run ID |
| `关联切片路径` | 题目相关切片路径列表或文本 |
| `related_slice_paths` | 关联切片路径的兼容字段 |
| `critic_basis_paths` | critic 依据路径兼容字段 |
| `参考母题全文` | 关联母题全文或拼接结果 |
| `mother_questions_full` | 结构化母题列表兼容字段 |

### 2.4 模板相关字段

| 字段 | 含义 |
| --- | --- |
| `模板任务` | 是否模板任务来源 |
| `模板正式题` | 是否模板正式入选题 |
| `模板备选题` | 是否模板备选题 |
| `模板父任务名称` | 模板父任务名 |
| `模板掌握度` | 模板槽位里的掌握度 |

### 2.5 离线 Judge 字段

| 字段 | 含义 |
| --- | --- |
| `offline_judge_score` | Judge 总分或融合分 |
| `offline_judge_decision` | `pass/review/reject` |
| `offline_judge_quality_score` | Judge 质量分 |
| `offline_judge_baseline_score` | Judge 基线分 |
| `offline_judge_quality_conclusion` | 质量分结论文本 |
| `offline_judge_baseline_conclusion` | 基线分结论文本 |

## 3. 列表接口返回特点

题库列表接口：

- 支持按 `keyword` 过滤题干
- 支持按 `material_version_id` 过滤教材版本
- 会为当前页题目补齐来源信息
- 返回时会额外加 `question_id` 作为当前索引

说明：

- `question_id` 在当前实现里本质上是题目在题库文件中的数组索引
- 删除题目后，后续题目的索引可能发生变化，因此不应把它当稳定业务主键长期保存

## 4. AI 调优接口

`POST /api/{tenant}/bank/optimize`

要求：

- `question_id` 必填
- `feedback` 必填

行为：

- 以当前题目 JSON 为基底
- 结合用户反馈请求 LLM 重写
- 优先解析 JSON
- JSON 解析失败时，再尝试从纯文本中抽取 `题干/选项/答案/解析`

如果调优结果里没有 `题干`，后端会报错，不会返回空题。

## 5. 手工更新接口

`POST /api/{tenant}/bank/update`

要求：

- `question_id` 必填
- `item` 必填
- `item.题干` 不能为空

补充规则：

- 如果请求里指定了 `material_version_id`，且新题目没带 `教材版本ID`，后端会自动补上

代码参考：

- [admin_api.py](/Users/panting/Desktop/搏学考试/AI出题/admin_api.py#L19239)

## 6. 批量新增接口

`POST /api/{tenant}/bank/add`

要求：

- `items` 必须是非空数组
- 每个题目至少要有 `题干`

补充规则：

- `question_id`、`_gen_key` 这类运行态字段会被剔除
- 可统一指定 `material_version_id` 给本批题目补教材版本

## 7. Excel 导出列定义

题库导出固定输出以下列：

| 导出列 | 含义 |
| --- | --- |
| `题干(必填)` | 题干 |
| `选项A(必填)` | 对应 `选项1` |
| `选项B(必填)` | 对应 `选项2` |
| `选项C` ~ `选项H` | 对应 `选项3` ~ `选项8` |
| `答案选项(必填)` | 大写答案选项 |
| `难度` | 数值难度 |
| `掌握程度` | 优先取 `模板掌握度`，否则取普通掌握程度字段 |
| `题型` | 题型/计算题口径 |
| `一级知识点` ~ `四级知识点` | 知识点层级 |
| `题目解析` | 解析 |
| `切片原文` | 主来源切片文本 |
| `关联切片数量` | 关联切片条数 |
| `关联切片路径` | 关联切片路径拼接文本 |
| `全部切片路径` | 主切片 + 关联切片路径全集 |
| `全部切片原文` | 所有切片原文拼接结果 |
| `参考母题全文` | 参考母题全文 |
| `结构化内容` | 结构化内容字段的字符串化结果 |
| `出题任务名称` | 来源任务名 |
| `出题任务ID` | 来源任务 ID |
| `出题RunID` | 来源 run ID |
| `离线Judge评分` | Judge 分数 |

## 8. 历史兼容字段说明

当前题库结构保留了较多兼容字段。

常见情况：

- `题干 / question / stem / 题目` 会被归一化到 `题干`
- `解析 / explanation` 会归一化到 `解析`
- `正确答案 / answer` 会归一化到 `正确答案`
- `选项` 字典、`options` 数组、`选项1~8` 都会尽量归一为 `选项1~8`

结论：

- 对外写文档时，建议统一按中文键说明
- 对内排查兼容问题时，再回看英文或旧键名

## 9. 常见误区

- `question_id` 不是稳定业务 ID，而是当前数组索引
- `offline_judge_score` 和 `offline_judge_quality_score` 不是同一个字段
- `关联切片路径` 不一定只含 1 条，导出时会被展开为多行文本
- `掌握程度` 导出优先级高于普通字段的是 `模板掌握度`
