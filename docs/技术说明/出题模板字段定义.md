# 出题模板字段定义

本文档说明“出题模板”对象的字段含义、默认值、校验规则，以及模板规划阶段的关键输出。

## 1. 模板用途

出题模板不是简单的“保存一组筛选条件”。

它同时约束三件事：

- 本次模板绑定哪一个教材版本
- 总题量如何按知识路径前缀拆分
- 总题量如何按 `掌握 / 熟悉 / 了解` 拆分

最终系统会把模板展开成一组“模板槽位”，每个槽位至少包含：

- `route_prefix`
- `mastery`
- `slice_id`

## 2. 模板字段

当前模板持久化字段如下：

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `template_id` | `string` | 模板唯一 ID |
| `name` | `string` | 模板名称，城市内唯一 |
| `description` | `string` | 模板备注说明，可为空 |
| `material_version_id` | `string` | 绑定教材版本 ID |
| `question_count` | `int` | 总题量 |
| `mastery_ratio` | `object` | 掌握度权重，固定三档：`掌握 / 熟悉 / 了解` |
| `mastery_percentages` | `object` | 根据权重换算出的百分比展示字段 |
| `route_rules` | `array` | 路由规则数组，每条规则描述一个知识路径前缀及其占比 |
| `route_total_ratio` | `number` | 所有路由权重之和，主要用于展示 |
| `created_at` | `string` | 创建时间 |
| `updated_at` | `string` | 更新时间 |

## 3. 前端默认值

管理后台创建模板时，默认值为：

- `question_count = 10`
- `mastery_ratio = { 掌握: 6, 熟悉: 3, 了解: 1 }`
- 路由规则默认先放 1 行空规则，需人工补全
- 教材版本下拉框只展示 `effective` 状态教材

这意味着模板是“只面向已生效教材”的出题规划工具，不支持直接绑定未生效版本。

## 4. 字段校验规则

### 4.1 `name`

- 不能为空
- 同一城市内大小写不敏感去重
- 更新模板时允许保留自己的原名称

失败示例：

- `模板名称不能为空`
- `模板名称已存在`

### 4.2 `material_version_id`

- 必填
- 必须能解析到当前城市存在的教材版本
- 前端通常只允许选择 `effective` 版本，但后端本身校验的是“版本存在”

失败示例：

- `请选择教材版本`
- `教材版本不存在`

### 4.3 `question_count`

- 必须能转成整数
- 最小值为 `1`
- 最大值为 `200`
- 超出范围时后端会自动夹紧到边界，不会保留原始非法值

失败示例：

- `题量必须是整数`

### 4.4 `mastery_ratio`

- 固定只认 `掌握 / 熟悉 / 了解`
- 每个值会被转成非负浮点数
- 允许单项为 `0`
- 但三项总和必须大于 `0`

说明：

- 模板不是直接存百分比，而是存“权重”
- 后端用最大余数法把总题量拆到三档掌握度

失败示例：

- `占比必须大于0`

### 4.5 `route_rules`

每条路由规则包含：

- `rule_id`
- `path_prefix`
- `ratio`

校验规则：

- 整体必须至少保留 1 条有效规则
- `path_prefix` 为空会被直接丢弃
- `ratio <= 0` 的规则会被直接丢弃
- `ratio` 会被转成浮点数
- 如果过滤后没有剩余有效规则，接口报错

失败示例：

- `至少需要配置一个切片路由占比`

## 5. 路由规则的实际含义

`route_rules` 不是精确百分比，而是“权重分配”。

例如：

```json
[
  { "path_prefix": "函数 > 一次函数", "ratio": 7 },
  { "path_prefix": "函数 > 二次函数", "ratio": 3 }
]
```

若 `question_count = 10`，则系统会优先把 10 道题拆成：

- 一次函数 7 题
- 二次函数 3 题

如果是 `question_count = 11`，则会通过最大余数法分配，保证总和仍等于 11。

## 6. 模板规划输出

模板真正执行前，会先构建模板规划结果，核心输出包括：

- `planned_slice_ids`
- `planned_slots`
- `route_breakdown`
- `mastery_ratio`
- `mastery_counts`

其中：

- `planned_slots` 表示实际待出题槽位
- 每个槽位包含 `slice_id + route_prefix + mastery`
- `route_breakdown` 用于前端或日志展示每个路径前缀下分到了多少题、候选切片有多少

## 7. 可达成性检查

模板在真正执行前，还会做一次 reachability 检查，确认模板目标是否“可达成”。

检查维度包括：

- 路由桶是否有候选切片
- 全局掌握度桶是否有候选切片
- 路由 x 掌握度组合桶是否存在缺口

如果某个桶完全没有候选切片，会返回类似错误：

- `函数 > 一次函数 没有可用 approved 切片`
- `全局缺少“熟悉”切片，无法满足模板占比`
- `路由桶[...] 需求... / 可用... / 预计缺口...`

这里的“可用切片”特指：

- 当前教材版本下
- 已 approved 的切片
- 且掌握度与路径前缀匹配

## 8. 常见误区

- 模板不是题目模板，而是“题量规划模板”
- 模板不直接绑定题干样式或题型文案
- 路由占比和掌握度占比都不是百分比字段，而是权重字段
- 模板保存成功不代表一定能出完题，还要看当前 approved 切片是否足够
- 模板续跑不是简单补剩余题量，而是按模板槽位缺口重建
