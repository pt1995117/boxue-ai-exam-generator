# Git 提交说明

这份说明只解决一件事：避免把运行态文件、缓存文件和业务代码混在同一个提交里。

## 提交原则

一个提交只做一类事情：

1. 基础设施治理
2. 业务功能改动
3. 测试补充或修正

不要把上面三类内容混在一个提交里。

## 允许提交的内容

通常可以提交：

- 源码文件
- 测试代码
- 配置模板
- 文档
- 启动脚本
- Git 忽略规则

## 不允许提交的内容

以下内容默认不应进入提交：

- `admin-web/node_modules/**`
- `admin-web/.vite/**`
- `.local/**`
- `logs/**`
- `data/admin_p0.db`
- `data/*/audit/**`
- `data/*/mapping/**`
- `data/*/slices/**`
- `data/*/bank/**`
- `data/*/exports/**`
- 运行生成的 `json/jsonl`
- 临时结果文件
- 导出物和素材，如 `xlsx/docx/pdf`
- 各类 `*.bak`、临时快照、调试转储

如果某个数据文件确实要版本化，必须先确认它属于“静态基线/样例”，而不是“运行结果”。

## 当前仓库的推荐拆提交方式

### 1. 基础设施治理提交

这类提交包括：

- 运行目录外置
- 缓存目录外置
- `.gitignore` 收紧
- Git hook
- 启动脚本和文档更新

### 2. 业务代码提交

这类提交包括：

- `admin-web/src/**`
- `admin_api.py`
- `exam_factory.py`
- `exam_graph.py`
- 相关测试文件

如果一次改动里既有仓库治理又有业务逻辑，先提治理，再提业务。

## 提交前检查

提交前至少执行一次：

```bash
git status --short
```

重点确认：

- 没有 `node_modules`
- 没有 `.vite`
- 没有 `.local`
- 没有 `data/*/audit`
- 没有数据库文件
- 没有导出物和素材文件

项目已配置提交前检查：

```bash
bash tools/install_git_hooks.sh
```

安装后，若暂存区包含运行态或缓存文件，提交会被阻止。

## 推荐提交流程

```bash
git status --short
git add <只属于同一类改动的文件>
git diff --cached --name-only
git commit -m "..."
```

不要使用无差别的：

```bash
git add -A
```

除非你已经明确确认暂存区里没有运行态文件、缓存和素材。

## 判断标准

如果一个文件满足下面任意一点，就先默认不要提交：

- 程序运行后会自动变化
- 只在你本机有意义
- 是缓存、数据库、日志或导出产物
- 是临时分析结果
- 是大体积二进制素材

拿不准时，先不要提交，先归类。
